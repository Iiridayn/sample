How to Proceed
  1. Download the lab11.c skeleton code.
  2. Read over the code until you are comfortable with what it does.
  3. Download ADD.mco and ADD.asm
  4. Open ADD.asm and familiarize yourself with the instructions. You need to know what the output/resulting data in registers/memory should be for those instructions.
  5. Compile the skeleton code.
  6. Run your program with debug turned on and ADD.mco as the filename. 
    Example:   ./lab11 1 ADD.mco
  7. It should output a TO-DO message and exit. The TO-DO message should specifically tell you which function you need to implement.
  8. Find the function in the skeleton code and implement it.
  9. Run the program again testing it to make sure that the function does what it is supposed to do
  10. Remove the TO-DO message and exit code from the function
  11. Run the program again.
  12. It should output another TO-DO message and exit.
  13. Repeat the above process until ADD.mco completely works. This means extensive testing!
  14. Once ADD.mco works, repeat the entire process again this time using AND.mco and AND.asm or other .mco files you create to implement other instructions.
  15. Repeat (kind of an annoying word isn't it) this process until you have implemented all of the functions in the skeleton code and all of the lc3 instructions.
  16. Finally (the definition of finally usually implies do this at the END) run your program with lab4.mco which is the copy of lab4 with which we will test your program. Test it to make sure it works. Note: TA's will not help you debug your program on lab4.mco if you have not already debugged your program using the shorter .mco files provided for you.
  17. Finally (#2) test your program using our copies of lab3, lab5 and lab6 to make sure your program works with those programs.
